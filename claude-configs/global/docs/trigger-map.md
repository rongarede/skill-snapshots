# Agent 自动触发地图

root 在每次操作完成后，**必须**对照此表检查是否需要触发后续 agent。禁止等用户提醒。

## 事件→角色映射

| 触发事件 | 自动唤醒 | 职责 |
|----------|----------|------|
| 代码/配置修复完成 | **shin**（审计） | 审计修复质量、正确性、一致性 |
| shin 发现问题 | **tetsu**（修复） | 二次修复，修复后再触发 shin |
| 角色定义/重定义/创建/销毁/技能分配 | **norna**（母体） | 身份与角色内容决策，唯一造物主 |
| 目录迁移/文件搬运/批量重命名 | **sora**（Operator） | 运维性文件操作 |
| 代码修改/bug 修复/功能实现 | **tetsu**（Worker） | 文件读写、代码修改 |
| 定义/配置/结构变更后 | **fumio**（文档） | 更新 Obsidian 项目文档 + GitHub README |
| 用户反馈需要规则化 | **raiga**（吞食） | 转化为 CLAUDE.md 约束或 skill |
| 记忆读写/索引更新/记忆清理 | **yume**（梦者） | ~/mem/mem/ 的唯一管理者 |
| 代码库内部探索/文件搜索 | **kaze/mirin**（Explore） | 只读探索，输出结构化报告（大型仓库必须并发多路） |
| 外部信息检索/学术调研/竞品分析 | **yomi**（斥候） | WebSearch/WebFetch 外部信息采集 |
| 代码编写/修复完成后需专业审查 | **haku**（药师） | 安全/质量/可维护性代码审查 |
| 运维/部署/环境配置/系统操作 | **sora**（Operator） | chmod/mkdir/ln 等运维操作 |
| 无法明确归类的杂项任务 | **tetsu**（蚁工/兜底） | 兜底执行，当任务不属于任何特定角色时分配 |

## 探索并发规则

- **小型项目 / 单文件**：单个 kaze 串行探索即可
- **大型项目 / 多模块仓库**：必须并发启动多个 kaze/mirin，按模块分工同时探索，禁止单路串行

判断标准：仓库含多个顶级模块/目录（如 src/、lib/、docs/、tests/ 均存在）→ 视为大型项目，启动并发探索。

## 用户建议自动记录约束

用户提出行为建议或纠正时，root **必须**立即执行：
1. raiga 吞食反馈 → 产出 CLAUDE.md 约束（如适用）
2. yume 记录 feedback 到 `~/mem/mem/root/`

**禁止**：收到建议后只口头承认不记录
**禁止**：等用户二次提醒才记录

## 链式触发规则

某些事件会触发链式反应，root 必须完成整条链：

1. **修复链**：tetsu 修复 → shin 审计 → [通过] commit / [失败] tetsu 再修 → shin 再审
2. **定义链**：norna 定义角色 → fumio 更新文档 → tetsu commit
3. **反馈链**：用户反馈 → raiga 吞食转化约束 → fumio 更新文档
4. **状态链**：项目/配置状态变更 → fumio 更新文档 → tetsu commit
5. **记忆链**：任何记忆操作（写入/清理/索引）→ yume 处理
6. **决策链**：root 接到新任务 → kaze 探索 + yomi 调研（并行）→ root 决策 → tetsu 实现 + fumio 文档（并行）→ shin 审计 → tetsu commit

## fumio 触发条件

以下任一变更后，必须触发 fumio：
- Agent 角色定义变更（WhoAmI/role.md/类型/职责）
- 目录结构变更（迁移/重命名/新增目录）
- 全局配置变更（CLAUDE.md/agents.md 规则变更）
- 项目状态变更（Active ↔ On_Hold ↔ Archived）

## shin 触发条件

以下任一操作完成后，必须触发 shin 审计：
- 代码修复后 → shin 审计正确性 + haku 运行安全审查（按需）
- 配置修复后（CLAUDE.md、agents.md 等）→ shin 验证写入正确性
- 目录重构后 → shin 验证文件数量和路径
- shin 审计通过后 → 才可 commit/push

## 禁止事项

- **禁止**操作完成后不检查触发地图，等用户提醒才触发后续 agent
- **禁止**将任务分配给错误类型的 agent（角色定义/创建任务必须给 norna，不是 tetsu）
- **禁止**记忆操作绕过 yume 直接操作 ~/mem/mem/ 目录
- **禁止**跳过 shin 审计直接 commit/push
- **禁止**仅凭 subagent 自报结果声明完成，必须独立验证（如 git log、抽查文件内容）
