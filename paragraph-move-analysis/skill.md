# 段落写作动作分析工具

对用户指定的论文段落进行逐句写作动作拆解，解释作者在每一句中"具体做了什么说服动作"，并给出可直接迁移的段落级仿写模板。

## 触发方式

- `/paragraph-analysis <论文名> <章节名>`
- 「分析这个段落的写作动作」
- 「逐句拆解这段 Introduction」

## 适用范围

本 Skill **不预设章节类型**，可用于：
- Introduction
- Discussion / Conclusion
- Related Work
- Methods
- Results
- 任何需要分析写作结构的段落

## 输入要求

- `{论文段落文本}` - 由用户明确指定
- `{论文名}` - 论文标题
- `{章节名}` - 段落所属章节

## 执行规范

### 1. 覆盖范围（强制）

- 对输入中的**每一个段落**分别分析
- 对段落中的**每一句话**进行逐句拆解
- 不得跳过句子
- 不得合并句子功能
- 不得假设该段落属于某一固定章节角色

### 2. 句子级写作动作拆解（核心）

对每个段落，按原文顺序编号句子（句1, 句2, …），对**每一句**输出：

| 字段 | 说明 |
|------|------|
| 句子编号 | 句1, 句2, ... |
| 简写 | 4-12 个汉字概括这句在段落中的作用（不是复述内容） |
| 写作动作类型 | 从下列选择 |
| 在做什么 | 用直白中文说明作者在"操作读者的哪一步理解" |
| 删掉会怎样 | 对逻辑连贯性、说服力、读者预期的具体破坏 |

**写作动作类型**：

| 类型 | 说明 |
|------|------|
| 背景/共识 | 建立背景或共识 |
| 聚焦/收缩 | 收缩讨论范围 |
| 对比/张力 | 制造对比或隐性冲突 |
| 具体缺口/问题 | 指出具体缺口或问题 |
| 策略/主张 | 提出立场、方法或判断 |
| 证据/支撑 | 提供支持性理由 |
| 防守/限定 | 提前防守或自我限定 |

### 3. 写作心法揭秘（段落级 · 强制）

每个段落分析结束后，必须输出：

- 作者在这个段落中是在铺路？转向？压缩问题？还是抬高自己/提前防守？
- 哪一句是这个段落中**最关键、最不能随意移动位置的句子**
- 如果改变句子顺序，读者会在哪一步产生困惑或怀疑

### 4. 完形填空式段落模板（强制）

每个段落必须输出 **1 组 3-5 句的英文完形填空模板**：

模板要求：
- 复现该段落的真实逻辑流
- 不是孤立单句
- 可直接迁移到其他论文

模板后必须逐条说明每个占位符应填入的**内容类型**：
- 已被广泛研究的研究方向
- 被忽略但关键的变量
- 特定应用场景
- 本文的方法或判断

## 语言约束

- 所有分析、解释、心法**必须使用中文**
- 仅以下内容允许使用英文：
  - 原文句子引用
  - 完形填空模板
  - 专有名词

**禁止使用**：
- "赋能""范式""底层逻辑""系统性""高阶""闭环"
- 只给术语不解释其在段落中的具体作用

## 禁止事项

- ❌ 内容复述（如"作者介绍了……"）
- ❌ 评价论文好坏
- ❌ 跨段合并分析
- ❌ 补写或猜测未提供的上下文

## 输出文件规范

**文件命名**：
```
{论文名}_{章节名}_提纲.md
```

**内容要求**：
- 可直接保存为 .md 文件
- 可作为论文写作提纲/仿写蓝本独立使用
- 不包含任何与分析无关的说明性文字

## 输出结构示例

```markdown
### 段落 P1

#### 句子级写作动作拆解

- 句1｜简写：建立研究热度｜动作类型：背景/共识｜在做什么：告诉读者这个领域很重要｜删掉会怎样：读者不知道为什么要关注这个话题
- 句2｜简写：指出现有局限｜动作类型：对比/张力｜在做什么：制造"虽然...但是..."的认知冲突｜删掉会怎样：读者不知道现有工作有什么问题
- ...

#### 写作心法揭秘

- 这个段落在**铺路 + 压缩问题**
- 最关键的句子是句3，因为它把宏观问题具体化为可操作的研究缺口
- 如果把句3移到句1位置，读者会困惑"为什么这个问题重要"

#### 完形填空式段落模板

In recent years, [A] has attracted growing attention in the context of [B]. However, existing studies tend to emphasize [C], leaving [D] insufficiently examined, particularly under [E]. To address this issue, this work focuses on [F], aiming to [G].

- [A]：已被广泛研究的研究方向
- [B]：特定应用场景或背景
- [C]：现有研究的主流关注点
- [D]：被忽略但关键的变量/维度
- [E]：使问题更具体的限定条件
- [F]：本文的研究焦点
- [G]：本文的目标或贡献
```

## 自检清单

- [ ] 每一句都有"简写 + 动作类型 + 在做什么 + 删掉后果"
- [ ] 解释了"如果删掉会出什么问题"
- [ ] 避免了行业黑话与空洞术语
- [ ] 给出了可直接迁移的段落模板
- [ ] 模板占位符有明确的内容类型说明
