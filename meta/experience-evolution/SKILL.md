# Experience Evolution

## 触发方式

- 会话结束时自动触发（Stop hook 提醒）
- 用户说「/evolve」或「总结经验」

## 当前模式：Observe Only

> 渐进式信任升级路径：**Observe** → Write Logs → Semi-auto → Cross-project
> 当前处于第一阶段，只观察不写入，生成摘要后自动保存。

## 观察维度

每次会话结束时，回顾并提取：

### 1. 行为模式
- root 做了哪些正确的决策？（强化）
- root 犯了哪些错误？（纠正）
- 哪些 subagent 表现好/差？（角色优化）

### 2. 工具效能
- 哪些 skill 被使用了？效果如何？
- 哪些 skill 应该被触发但没有？
- 工具链中有无断裂（手动补了自动化该做的事）？

### 3. 知识缺口
- 会话中遇到了哪些未知？
- 哪些知识应该被持久化到 CLAUDE.md 或 docs/？
- 哪些 feedback 应该被提升为正式规则？

### 4. 流程改进
- 有无重复性工作可以自动化？
- 工作流中有无瓶颈？
- 角色分工是否合理？

## 输出格式

```markdown
## Session 经验摘要

### 强化
- [正确做法 1]
- [正确做法 2]

### 纠正
- [错误 1] → [正确做法]
- [错误 2] → [正确做法]

### 知识新增
- [新发现 1]
- [新发现 2]

### 建议改进
- [改进建议 1]（优先级：高/中/低）
- [改进建议 2]
```

## 保存规则

Observe 模式下：
1. 生成经验摘要 → 展示给用户
2. 自动由 yume 保存到 `~/mem/mem/agents/root/`
3. 高优先级建议 → 自动提议写入 CLAUDE.md（root 自主判断后执行）
4. 低优先级建议 → 记录到 memory，Sunday Rule 时集中处理

## 未来升级路径

| 阶段 | 触发 | 行为 | 前置条件 |
|------|------|------|---------|
| Observe | 手动 | 生成摘要后自动保存 | 当前 |
| Write Logs | 自动 | 自动写入 memory，不改 CLAUDE.md | 积累 20+ 条无误经验 |
| Semi-auto | 自动 | 自动写入 memory + 提议 CLAUDE.md 变更 | 积累 50+ 条，误报率 <5% |
| Cross-project | 自动 | 跨项目迁移经验 | 多项目验证 |
